Praticando um pouco mais

O que represenda a letra M e a letra U ao lado do nome do arquivo

  • M: A letra M representa o estado Modified, do português modificado. Isso significa que o arquivo já existia no repositório, mas que recebeu alguma modificação que ainda não foi registrada no Git.
  • U: A letra U representa o estado Untracked, do português não rastreado. Isso significa que o arquivo ainda não existia no repositório e que ainda não teve seu registro (commit) feito no Git.

Git commit

O comando git commit é usado para registrar as alterações feitas em seus arquivos no repositório Git.
Cada commit representa um ponto na história do seu projeto e inclui as alterações feitas desde o último commit. Ao fazer um commit, você está criando uma "captura" das alterações no seu projeto, tornando-as permanentes no histórico.
Todo commit conta um um id único e traz uma referência aos commits anteriores. Assim, através dessa cadeia de commits, o Git registra um histórico completo de todos os commits realizados no repositório.

Sintaxe

git commit -m "Mensagem do commit"

Mensagem no commit

  • Devem ser simples e objetivas, garanta que suas mensagens de commit tenham clareza e sejam suficientemente descritivas
  • Mantenha a mensagem curta e concisa: A primeira linha da mensagem deve conter, no máximo, 72 caracteres
  • Uso de verbo no infinitivo(adicionar, corrigir, atualizar)
  • Evite detalhes técnicos: Não inclua detalhes técnicos complexos na mensagem de commit. Esses detalhes podem ser adicionados nos comentários de código ou na documentação.

Quando usar Git commit?

O comando git commit é usado quando você deseja salvar as alterações feitas em seus arquivos e registrar essas mudanças como um novo ponto na história do seu projeto.
Você deve usar git commit após realizar alterações nos seus arquivos e adicionar essas alterações ao "index" (também conhecido como "staging area") usando o comando git add.
